Architecture

How the system fits together.

The key idea is to turn fragmented on-chain lending data into a structured, monetizable risk layer. The frontend is only the visible surface; the real product is the data pipeline plus analysis layer behind it.

01 · Index

Custom subgraphs

Each chain is indexed separately so positions, health factors, alerts and liquidations can be queried quickly instead of reading raw blockchain events every time.

02 · Analyze

Risk engine

The backend aggregates subgraph data into wallet-level risk, systemic risk, cascade scenarios and liquidation exposure indicators.

03 · Monetize

Dashboard + paid APIs

Stripe supports SaaS subscriptions. Ampersend could support API-level monetization for premium endpoints and agent workflows.

Ampersend fit

From SaaS subscription to programmable paid API access.

Stripe works well for dashboard subscriptions. Ampersend could become interesting if the product evolves into a paid API where developers, funds, bots or agents pay for precise data endpoints instead of only accessing a web dashboard.
